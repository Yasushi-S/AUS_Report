## [2026-06-17] 修正指示 #1 ── 起動時コンソール出力の文字化け修正

### 改修の背景・目的
起動時の案内メッセージ（日本語）が、実行環境によって標準出力のエンコーディングが異なるため文字化けすることがある。標準出力・標準エラーをUTF-8に固定し、どの環境からアプリを起動しても表示が崩れないようにする。

### 影響範囲
- 対象ファイル: app.py
- 影響する既存機能: なし（起動時のコンソール出力のみ。アプリの動作・APIには影響しない）
- DBスキーマ変更: なし

### 修正内容詳細
- app.py の先頭付近で `sys.stdout` / `sys.stderr` を UTF-8（`errors='replace'`）に再設定する
- コンソールが存在しない環境（タスクスケジューラ等）で `reconfigure` が失敗しても起動が止まらないよう例外を握る
- 既存の print 文・起動ロジックは変更しない

### 注意事項
- 表示のみの修正であり、デグレードリスクは低い

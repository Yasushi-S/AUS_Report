## [2026-06-17] 修正指示 #1 ── 起動時コンソール出力の文字化け修正

### 改修の背景・目的
起動時の案内メッセージ（日本語）が、実行環境によって標準出力のエンコーディングが異なるため文字化けすることがある。標準出力・標準エラーをUTF-8に固定し、どの環境からアプリを起動しても表示が崩れないようにする。

### 影響範囲
- 対象ファイル: app.py
- 影響する既存機能: なし（起動時のコンソール出力のみ。アプリの動作・APIには影響しない）
- DBスキーマ変更: なし

### 修正内容詳細
- app.py の先頭付近で `sys.stdout` / `sys.stderr` を UTF-8（`errors='replace'`）に再設定する
- コンソールが存在しない環境（タスクスケジューラ等）で `reconfigure` が失敗しても起動が止まらないよう例外を握る
- 既存の print 文・起動ロジックは変更しない

### 注意事項
- 表示のみの修正であり、デグレードリスクは低い

## [2026-06-17] 修正指示 #2 ── ブラウザの言語誤判定（中国語表示）の修正

### 改修の背景・目的
ブラウザでアクセスすると、Google翻訳バーが「中国語（繁体字）」を最初の候補として表示してしまう。`templates/index.html`の`<html>`タグに`lang`属性が指定されておらず、ブラウザがページ内容（漢字主体）から言語を自動判定して中国語と誤認識していたことが原因。

### 影響範囲
- 対象ファイル: templates/index.html, templates/edit.html, templates/error.html
- 影響する既存機能: なし（表示のみ）
- DBスキーマ変更: なし

### 修正内容詳細
- 各テンプレートの `<html>` を `<html lang="ja">` に変更する

### 注意事項
- 表示のみの修正であり、デグレードリスクは低い

## [2026-06-18] 修正指示 #3 ── ポート番号変更（50013 → 50003）

### 改修の背景・目的
運用上の都合により、アプリの待受ポート番号を 50013 から 50003 に変更する。

### 影響範囲
- 対象ファイル: config.py
- 影響する既存機能: アプリのアクセスURL（http://[PCのIPアドレス]:50013 → :50003）が変わる。Windowsファイアウォールでポート許可設定をしている場合は再設定が必要。
- DBスキーマ変更: なし

### 修正内容詳細
- config.py の `PORT = 50013` を `PORT = 50003` に変更する

### 注意事項
- Windowsファイアウォールで TCP 50013 を許可している場合、TCP 50003 への許可ルールへの更新が必要
- 院内端末側のブックマーク・アクセスURLも 50003 に更新が必要
